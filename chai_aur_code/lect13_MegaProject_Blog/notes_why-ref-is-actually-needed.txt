
MY DOUBT :::: ref provide krte hi kyu h 



Yehi actual question hai. 😄

Sabse pehle samjho **React ka rule**:

> React me hum normally **DOM ko directly touch nahi karte**. React khud DOM ko manage karta hai.

Lekin kuch situations me hume **actual HTML element ko directly access** karna padta hai. Uske liye **`ref`** use hota hai.

---

## Example 1: Input ko focus karna

Without `ref`:

```jsx
<input />
```

Ab agar button click hote hi cursor input me lana ho, to kaise karoge?

`ref` use karte hain:

```jsx
const inputRef = useRef();

<input ref={inputRef} />

<button onClick={() => inputRef.current.focus()}>
    Focus
</button>
```

Yahan `ref` ki wajah se actual `<input>` mil gaya.

---

## Example 2: Input ki value lena

```jsx
const inputRef = useRef();

<input ref={inputRef} />

<button onClick={() => {
    console.log(inputRef.current.value);
}}>
    Show Value
</button>
```

---

## Example 3: Video play/pause

```jsx
const videoRef = useRef();

videoRef.current.play();
videoRef.current.pause();
```

---

## Example 4: Scroll karna

```jsx
divRef.current.scrollIntoView();
```

---

# To `ref` provide hi kyu karte hain?

> **Taaki parent component actual DOM element ko directly access ya control kar sake jab normal React props/state se kaam na ho.**

---

## Flow

```text
Without ref

Component
    │
    ▼
<input>

❌ Input ko directly control nahi kar sakte


With ref

Component
    │
    ▼
ref
    │
    ▼
<input>

✅ focus()
✅ value
✅ play()
✅ pause()
✅ scrollIntoView()
```

---

### 📝 Short Notes

**Why do we provide `ref`?**

> **A `ref` is provided to get direct access to a DOM element so that we can perform operations like focusing an input, reading its value, scrolling, playing media, or calling other DOM methods that cannot be done through props or state alone.**

### Easy Line to Remember ⭐

> **`ref` is like a shortcut to directly access and control an HTML element.**
