# Sabse Important Line (Interview Answer) ⭐

> **Context API was introduced to solve prop drilling, but it is not a complete state management solution. In large applications, Context can cause unnecessary re-renders, become difficult to organize, and lacks features like middleware, DevTools, and predictable state updates. Redux Toolkit provides scalable, organized, and efficient global state management, which is why it is preferred for medium to large applications.**

## 💡 Ek line me yaad rakhna:

* **Context API = Data share karne ka tool.**
* **Redux Toolkit = Large applications ke liye complete state management system.**

**Prop drilling solve karna Context API ka goal tha. Large-scale state management solve karna Redux Toolkit ka goal hai.** Yehi dono ke beech ka sabse bada difference hai.



# Lekin cart to same tha?

Haan.

Cart array same ho sakta hai.

```
cart
```

same reference bhi ho sakta hai.

Lekin

```jsx
value={{
    theme,
    cart
}}
```

Ye outer object har render me naya ban raha hai.

Jaise

```js
const value1 = {
    theme: "light",
    cart
}

const value2 = {
    theme: "dark",
    cart
}
```

`cart` same hai,

lekin `value1` aur `value2`

alag objects hain.

```
Same
```

```
theme
```

## 🔥 Ek chhota JavaScript example

Is code ko browser console me chalao:

```js
const cart = [];

const value1 = {
  theme: "light",
  cart
};

const value2 = {
  theme: "dark",
  cart
};

console.log(value1 === value2);        // false
console.log(value1.cart === value2.cart); // true
```

**Output:**

```
false
true
```

Yani:

* `cart` **same** hai.
* Lekin `value` object **naya** hai.

Isi wajah se Context Provider ko lagta hai ki uski `value` change ho gayi.
